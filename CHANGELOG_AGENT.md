# CHANGELOG_AGENT

## 2026-02-22 (UI/実装担当対応)

### 目的
- 記事本文の大量追加より、**構造と見やすさ**を優先してMVP UIを改善
- 現状の「見にくい / 分かりにくい」状態を、レイアウト・視認性・導線で修正

### 実施内容
1. **全体レイアウト改善**
   - `max-w-5xl` → `max-w-6xl` で可読幅を最適化
   - ヘッダーを sticky 化し、主要導線（記事一覧/運営方針/広告開示）を常時アクセス可能に
   - 余白・文字サイズ・コントラストを見直し

2. **トップページUI改善**
   - ヒーロー、比較ブロック、注目記事のカードを統一デザイン化
   - CTAを視認しやすいボタンUIへ変更（`#` リンクを実ページ `/articles` に修正）
   - 比較表のヘッダー・セル構造を整理し、可読性を向上

3. **記事一覧UI改善**
   - 一覧導入ヘッダーを新設
   - カテゴリバッジを可視化
   - カード型一覧 + 明確な「記事を読む」ボタンへ変更

4. **記事詳細UI改善 + 表示不具合修正**
   - `params` 型を Promise ではなく通常オブジェクトに修正（Next App Router標準）
   - 見出し階層・本文行間・余白を調整して読みやすさ改善
   - 右カラムに目次/広告開示ボックスを追加（PC）
   - PR開示表記を維持

5. **グローバルCSS整備**
   - `card` / `badge` / 可読性補助クラスを追加
   - body・リンク・画像のベーススタイルを明確化

### 検証
- `npm run build` 成功
- 静的出力（`output: export`）でページ生成を確認

### 備考
- 誇大表現は追加せず、既存の広告開示方針を維持
