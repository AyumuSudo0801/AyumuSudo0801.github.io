### 観察した共通パターン
- **トップ**：検索起点（地域・目的）+ 社会的証明（掲載数/口コミ数/実績）
- **カテゴリ**：ユーザーの意思決定軸（料金・目的・学年・エリア）で絞り込む
- **記事**：結論先出し + 比較表 + 詳細解説の3層構造
- **比較表**：ファーストビュー直下か目次直下に配置（離脱抑制）
- **CTA**：本文途中・記事末・比較表下に複数配置（同一文言で反復）
- **フォーム**：入力負荷を下げる（段階入力、必須最小化、安心訴求）

## 3. 採用するUI原則（丸パクリ禁止・抽象化）

1. **意図先行ナビゲーション**：最初に「目的」「対象者」「予算」「地域」を選ばせる
2. **結論先出し**：記事冒頭で「おすすめ上位3件 + 理由」を提示
3. **比較ファースト**：比較表を早い位置に置き、詳細はアコーディオン/タブで後段化
4. **CTA一貫性**：文言・色・配置を統一（迷わせない）
5. **信頼証拠の分離表示**：PR表記、評価ロジック、更新日、監修者を常時可視化
6. **フォーム摩擦最小化**：2〜3ステップ化、入力途中保存、エラー即時表示
7. **モバイル起点**：1カラム前提、親指領域にCTA固定
8. **アクセシビリティ標準化**：コントラスト、見出し階層、focus可視、alt/aria徹底

## 4. ページ別ワイヤーフレーム要件（文章）

### 4-1. トップページ
- Hero：検索（目的/学年/地域）
- 直下：人気比較カテゴリ（カード）
- 信頼ブロック：掲載基準・監修・更新日
- 人気ランキング（3〜5件）
- 比較記事導線（目的別）
- FAQ
- 最終CTA（無料診断/資料請求）

### 4-2. カテゴリ一覧ページ
- 上部フィルタ：料金帯・形式・対象・オンライン可否
- 並び替え：おすすめ/安い順/評価順
- 比較カード一覧（固定ヘッダで項目名維持）
- サイド/下部CTA：一括比較・無料相談

### 4-3. 記事ページ（比較・ランキング）
- 冒頭要約（結論）
- 比較表（sticky列名）
- 詳細レビュー（メリデメ、向く人/向かない人）
- 選び方セクション
- よくある質問
- CTA（本文中2回 + 末尾1回）

### 4-4. 比較表コンポーネント
- 列：サービス名/料金/月額換算/対象/特徴/無料体験
- 行：最大5〜8件（多すぎる場合はページング）
- モバイル：横スクロール + 重要列固定 + 比較チェック機能

### 4-5. フォームページ
- Step1 条件入力 → Step2 連絡先 → Step3 確認
- 安心要素：個人情報説明、所要時間、離脱時保存
- 完了後：次アクション（LINE相談/電話予約）

## 5. CSS設計方針（命名・レイヤー・保守性）

- **設計思想**：CUBE CSS + Design Token
- **命名**：`c-`(component), `l-`(layout), `u-`(utility), `is-`(state)
  - 例：`c-card`, `l-container`, `u-mt-16`, `is-active`
- **レイヤー**：
  1. `tokens.css`（色/余白/タイポ/影/z-index）
  2. `base.css`（reset, element）
  3. `layout.css`（グリッド/コンテナ）
  4. `components.css`（カード/比較表/CTA）
  5. `utilities.css`（単機能）
- **保守ルール**：
  - 1コンポーネント1責務（過剰ネスト禁止）
  - 色・余白の直書き禁止（token参照）
  - 重要UIはStorybook相当で状態管理（default/hover/focus/error）

## 6. MVPで作るべきページ/コンポーネント/実装順（最優先）

### 6-1. MVP対象ページ（必須4ページ）
1. **トップページ**（検索起点 + 信頼訴求 + 人気比較導線）
2. **カテゴリ一覧ページ**（条件絞り込み + 比較カード一覧）
3. **比較記事ページ**（結論先出し + 比較表 + 詳細解説 + CTA）
4. **問い合わせフォームページ**（3ステップ）

> まずは上記4ページだけで「流入→比較→送客」の導線を成立させる。

### 6-2. MVP必須コンポーネント（優先度付き）

**P0（初週で実装）**
- `c-header`（グロナビ、モバイルメニュー）
- `c-hero-search`（目的/学年/地域）
- `c-category-card`（カテゴリ導線）
- `c-ranking-card`（上位3〜5件表示）
- `c-cta-button`（主/副の2種）
- `c-footer`

**P1（2週目で実装）**
- `c-filter-bar`（料金/形式/対象/オンライン）
- `c-service-card`（ロゴ、特徴、価格、CTA）
- `c-compare-table`（stickyヘッダ、モバイル横スクロール）
- `c-procon`（向く人/向かない人）
- `c-faq-accordion`
- `c-pr-disclosure`（PR表記、評価ロジック、更新日）

**P2（3週目で実装）**
- `c-step-form`（Stepインジケータ付き）
- `c-input`, `c-select`, `c-validation-message`
- `c-fixed-mobile-cta`（モバイル下部固定）
- `c-breadcrumb`

### 6-3. 実装順序（3週間の具体計画）

#### Week 1: 基盤 + トップページ成立
1. Design Token定義（color/type/spacing/radius/shadow）
2. CSSレイヤー作成（tokens/base/layout/components/utilities）
3. 共通レイアウト（header/footer/container）
4. トップページのP0コンポーネント実装
5. モバイル表示確認（375px基準）

#### Week 2: 比較体験の中核実装
1. カテゴリ一覧ページ実装（filter + service card）
2. 比較記事ページ実装（要約→比較表→詳細→FAQ）
3. 比較表のレスポンシブ最適化（横スクロール、列優先表示）
4. PR表記・更新日・監修情報の常設化

#### Week 3: フォーム完了 + 品質調整
1. 3ステップフォーム実装（条件→連絡先→確認）
2. 入力バリデーション（即時/送信時）
3. 固定CTA・パンくず導入
4. UI監査（可読性、タップ領域、focus、見出し階層）
5. 文言・余白・状態差分の最終調整

## 7. 改善フェーズ（MVP後）
1. Core Web Vitals改善（画像最適化、遅延読み込み、JS分割、フォント最適化）
2. アクセシビリティ監査の深掘り（スクリーンリーダー読上げ順、aria属性再点検）
3. 比較表の拡張（並べ替え、比較チェック、保存）
4. 構造化データ（FAQ/Review/Breadcrumb）拡充
5. 検索意図別テンプレ拡張（初心者向け、料金重視、短期成果重視）
